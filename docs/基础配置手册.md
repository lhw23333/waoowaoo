## 一、系统环境要求

| 组件 | 最低要求 | 推荐版本 |
|------|---------|---------|
| Node.js | >= 18.18.0 | 22.x LTS |
| MySQL | 8.0+ | 8.4.x |
| Redis | >= 5.0.0 | 7.x |
| 操作系统 | Windows 10+ / Linux / macOS | Ubuntu 22.04 / Windows 11 |
| 内存 | 4 GB | 8 GB+ |
| 磁盘 | 10 GB | 50 GB+（视频素材占用大） |

```
┌──────────────────────────────────────────────────┐
│              Waoowaoo 服务组成                      │
├──────────────────────────────────────────────────┤
│  [1] Next.js 服务 (端口 3000)    ← Web 前端 + API  │
│  [2] Image Worker               ← 图片生成队列    │
│  [3] Video Worker               ← 视频生成队列    │

│  [5] Text Worker                ← LLM 文本分析队列 │
│  [6] Watchdog                   ← 任务健康监控    │
│  [7] Bull Board (端口 3010)      ← 队列管理面板    │
├──────────────────────────────────────────────────┤
│  MySQL 8.0 (端口 3306)           ← 数据持久化      │

```

## 三、环境变量配置详解

环境变量文件位于项目根目录 `.env`。以下按模块详细说明：

### 3.1 数据库配置

```bash
DATABASE_URL="mysql://root:your_password@localhost:3306/waoowaoo"
```

- 格式：`mysql://用户名:密码@主机:端口/数据库名`
- 确保 MySQL 已创建 `waoowaoo` 数据库：`CREATE DATABASE waoowaoo CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;`

### 3.2 存储配置

```bash
STORAGE_TYPE=local          # local（本地）或 cos（腾讯云COS）
```

**本地存储**（默认推荐）：
- 文件存储在 `./data/uploads/` 目录
- 无需额外配置，开箱即用

**腾讯云 COS 存储**（可选，适合生产环境）：
```bash
STORAGE_TYPE=cos
COS_SECRET_ID=你的SecretId
COS_SECRET_KEY=你的SecretKey
COS_BUCKET=你的存储桶名称
COS_REGION=ap-guangzhou        # 地域，如 ap-beijing、ap-shanghai
```

### 3.3 认证与安全密钥

```bash
NEXTAUTH_URL=http://localhost:3000    # 你的访问地址
NEXTAUTH_SECRET=随机字符串              # 用于 JWT 签名，必须修改！

CRON_SECRET=随机字符串                  # 定时任务验证密钥
INTERNAL_TASK_TOKEN=随机字符串          # 内部任务通信令牌
API_ENCRYPTION_KEY=随机字符串           # API Key 的 AES 加密密钥
```

> 生成随机密钥：`node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"`

> **安全提示**：`API_ENCRYPTION_KEY` 用于加密存储你在平台中配置的所有 AI 服务 API Key。一旦更改，已保存的 Key 将无法解密，需要重新配置。

### 3.4 Redis 配置

```bash
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
REDIS_USERNAME=               # 无密码时留空
REDIS_PASSWORD=               # 无密码时留空
REDIS_TLS=                    # 启用 TLS 时填 true
```

### 3.5 Worker 并发配置

```bash
QUEUE_CONCURRENCY_IMAGE=50    # 图片生成并发数
QUEUE_CONCURRENCY_VIDEO=50    # 视频生成并发数
QUEUE_CONCURRENCY_VOICE=20    # 语音合成并发数
QUEUE_CONCURRENCY_TEXT=50     # 文本分析并发数
```

> 建议根据 AI 服务商的 API 速率限制适当调低。例如免费 API 建议设为 5-10。

### 3.6 任务监控配置

```bash
WATCHDOG_INTERVAL_MS=30000       # 看门狗检查间隔（毫秒）
TASK_HEARTBEAT_TIMEOUT_MS=90000  # 任务心跳超时（毫秒）
```

- Watchdog 会自动恢复卡住的任务、清理超时的僵尸任务
- 超时任务会自动重试（最多 5 次）

### 3.7 Bull Board 管理面板

```bash
BULL_BOARD_HOST=0.0.0.0
BULL_BOARD_PORT=3010
BULL_BOARD_BASE_PATH=/admin/queues
BULL_BOARD_USER=admin            # 管理面板用户名
BULL_BOARD_PASSWORD=admin123     # 管理面板密码
```

访问地址：`http://localhost:3010/admin/queues`

### 3.8 日志配置

```bash
LOG_UNIFIED_ENABLED=true
LOG_LEVEL=INFO                   # DEBUG / INFO / WARN / ERROR
LOG_FORMAT=json                  # json 或 text
LOG_DEBUG_ENABLED=false          # 开启详细调试日志
LOG_AUDIT_ENABLED=true           # 开启审计日志
LOG_REDACT_KEYS=password,token,apiKey,authorization,cookie,secret
```

### 3.9 计费配置

```bash
BILLING_MODE=OFF                 # OFF / SHADOW / ENFORCE
```

| 模式 | 说明 |
|------|------|
| `OFF` | 关闭计费（默认，个人使用推荐） |
| `SHADOW` | 影子模式，记录消费但不扣费 |
| `ENFORCE` | 强制模式，余额不足时拒绝任务 |

## 四、AI 服务 API 配置指南

启动应用后，进入 **用户头像 → 个人设置 → API 配置** 页面进行配置。

### 4.1 配置提供商（Provider）

系统内置以下提供商，需要填入对应的 API Key：

| 提供商 | 用途 | API Key 获取地址 | 说明 |
|--------|------|-----------------|------|
| **OpenAI Compatible** | LLM 文本分析 | 取决于所选服务 | 默认 BaseURL: `https://openrouter.ai/api/v1` |
| **fal.ai** | 图片/视频生成 | https://fal.ai/dashboard/keys | 支持 Banana、Wan、Kling、Sora 等 |
| **Google AI** | 图片/视频生成 | https://aistudio.google.com/apikey | Gemini 图片、Imagen、Veo |
| **火山引擎 (Ark)** | 图片/视频生成 | https://console.volcengine.com/ | Seedream、Seedance |
| **阿里百炼 (Qwen)** | 语音合成 TTS | https://dashscope.console.aliyun.com/ | 千问 TTS 多角色配音 |
| **MiniMax** | 视频生成 | https://platform.minimaxi.com/ | 海螺视频 |
| **Vidu** | 视频生成 | https://vidu.com/ | Vidu 视频模型 |

### 4.2 推荐最小配置

只需配置以下 3 个 API Key 即可体验完整流程：

| API Key | 用途 | 推荐服务 |
|---------|------|---------|
| **LLM API Key** | 剧本分析、角色提取、分镜规划 | OpenRouter（聚合多家 LLM） |
| **图片 API Key** | 角色/场景/分镜图片生成 | fal.ai（Banana 模型）或 Google AI（Gemini） |
| **视频 API Key** | 分镜视频生成 | fal.ai 或 火山引擎 |

**可选**：
- 阿里百炼 API Key → 启用 AI 配音
- fal.ai / Vidu API Key → 启用口型同步

### 4.3 配置模型

在 API 配置页添加提供商后，进入 **模型管理** 添加具体模型：

**LLM 文本模型**（推荐）：
| 模型 | Provider | Model ID |
|------|----------|----------|
| Gemini 2.5 Flash | google | gemini-2.5-flash-preview |
| DeepSeek V3 | openai-compatible | deepseek/deepseek-chat (via OpenRouter) |
| Claude Sonnet | openai-compatible | anthropic/claude-sonnet-4 (via OpenRouter) |

**图片模型**（推荐）：
| 模型 | Provider | Model ID |
|------|----------|----------|
| Banana 2 | fal | fal-ai/nano-banana-2 |
| Gemini 3 Pro Image | google | gemini-3-pro-image-preview |
| Seedream 4.5 | ark | doubao-seedream-4.5 |

**视频模型**（推荐）：
| 模型 | Provider | Model ID |
|------|----------|----------|
| Wan 2.6 | fal | fal-ai/wan/v2.6/1080p |
| Seedance 1.5 Pro | ark | doubao-seedance-1.5-pro |
| Kling 3 Pro | fal | fal-ai/kling-video/v3/pro |
| Veo 3.1 Fast | fal | fal-ai/veo3.1/fast |

### 4.4 项目级模型绑定

在每个项目的 **设置** 中，可以为以下环节指定使用哪个已注册的模型：

| 设置项 | 用途 |
|--------|------|
| 分析模型 (analysisModel) | 剧本分析、角色提取、分镜规划等文本任务 |
| 角色模型 (characterModel) | 角色参考图生成 |
| 场景模型 (locationModel) | 场景参考图生成 |
| 分镜模型 (storyboardModel) | 分镜面板图片生成 |
| 编辑模型 (editModel) | 图片修改/编辑 |
| 视频模型 (videoModel) | 视频片段生成 |

## 五、启动与停止服务

### 5.1 启动全部服务

```bash
cd waoowaoo
npm run start                # 生产模式（需先执行 npm run build）
# 或
npm run dev                  # 开发模式（支持热更新）
```

### 5.2 单独启动各组件

```bash
npm run start:next           # 仅启动 Next.js Web 服务
npm run start:worker         # 仅启动 4 个 Worker 进程
npm run start:watchdog       # 仅启动 Watchdog 监控
npm run start:board          # 仅启动 Bull Board 管理面板
```

### 5.3 停止服务

Windows：
```bash
taskkill /F /IM node.exe     # 停止所有 Node 进程
```

### 5.4 数据库操作

```bash
npx prisma db push           # 同步数据库 Schema（首次或更新后）
npx prisma generate          # 重新生成 Prisma Client
npx prisma studio            # 启动数据库可视化管理界面
```

## 七、常见问题排查

### Q1: 页面卡顿/SSE 连接受限
HTTP 模式下浏览器限制并发连接数。安装 [Caddy](https://caddyserver.com/) 启用 HTTPS：
```bash
caddy run --config Caddyfile
```
然后访问 `https://localhost:1443`

### Q2: Redis 版本警告 "recommended 6.2.0"
Redis 5.0+ 即可正常运行，警告不影响功能。如需消除警告，升级至 Redis 7.x。

### Q3: Worker 报错 "Redis version needs to be >= 5.0.0"
当前 Redis 版本过低（如 3.x），BullMQ 要求最低 Redis 5.0。请升级 Redis。

### Q4: 任务一直 "queued" 不执行
1. 检查 Worker 进程是否运行：查看日志中 `[Workers] ready` 消息
2. 检查 Redis 连接：`redis-cli ping` 应返回 `PONG`
3. 查看 Bull Board 面板确认队列状态

### Q5: 图片/视频生成失败
1. 进入 **个人设置 → API 配置** 检查对应 API Key 是否正确
2. 使用 "测试连接" 功能验证 API 可达性
3. 检查 Bull Board 中对应队列的 Failed 任务，查看错误详情

### Q6: 数据库迁移 / 更新后启动报错
```bash
git pull                     # 拉取最新代码
npm install                  # 更新依赖
npx prisma db push           # 同步数据库变更
npm run build                # 重新构建
npm run start                # 启动
```
