## 二、技术架构

### 2.1 核心设计思想

VAST 将文本到视频生成拆解为两个独立阶段，模拟影视制作中"先画分镜、再拍摄"的工作流 **[S1][S3]**：

```
文本输入 → [StoryForge] → 故事板 → [VisionForge] → 视频输出
```

论文原文表述：*"a two-stage framework that bridges the gap between textual input and high-quality video synthesis"* **[S3]**

### 2.2 第一阶段 StoryForge

**功能**：将文本描述转换为结构化故事板 **[S1][S3]**

| 组件 | 说明 | 来源 |
|------|------|------|
| 姿态自编码器 | 通过编码-解码+重建损失，学习人物姿态的潜在表示 | **[S1]** |
| 布局自编码器 | 同上范式，学习物体空间布局的潜在表示 | **[S1]** |
| 因果多模态 LLM | 接收编码器输出+文本 token，自回归预测各时步的姿态和布局 | **[S1]** |

故事板的输出内容：*"human poses and object layouts to represent the structural essence of the scene"*（人物姿态和物体布局，表征场景的结构本质）**[S1]**

### 2.3 第二阶段 VisionForge

**功能**：基于故事板生成最终视频 **[S1][S3]**

| 组件 | 说明 | 来源 |
|------|------|------|
| DiT 扩散模型 | 扩散变换器架构，扩展以处理多种输入类型 | **[S1]** |
| 故事板适配器 | 自注意力层融合多模态特征，生成统一表示后注入 DiT | **[S1]** |
| 外观条件注入 | 结合文本描述和目标主体外观信息，实现跨片段角色一致 | **[S1][S3]** |

论文原文表述：*"producing high-quality videos with smooth motion, temporal consistency, and spatial coherence"* **[S1]**

### 2.4 训练策略

三阶段训练流程 **[S1]**：

| 阶段 | 操作 | 数据 |
|------|------|------|
| 1 | 单独训练 StoryForge | 完整视频数据集 |
| 2 | 冻结 StoryForge 编码器，训练 VisionForge | 视频+图像混合 |
| 3 | 联合微调，随机混合真实/生成故事板 | 混合数据 |

### 2.5 训练数据

- **3000万+** 高质量视频片段 + **1亿+** 高质量图像 **[S1]**
- 采样率：每秒 5 帧 **[S1]**
- 标注：人物姿态、边界框、图像描述 **[S1]**

## 三、评测结果

### 3.1 关于 VBench

VBench 是由南洋理工大学 S-Lab、上海人工智能实验室等机构开发的视频生成评测基准，发表于 **CVPR 2024（Highlight）**，包含 **16 个评估维度**，涵盖时间质量、帧一致性、语义对齐等，是当前视频生成领域公认的权威自动化评测 **[S4]**。

### 3.2 VAST 评测成绩

以下数据直接来自论文 **[S1]**：

| 维度 | VAST | 第二名 |
|------|------|--------|
| **总分** | **89.71** | 83.78 |
| **质量分** | **88.98** | 84.39 |
| **语义分** | **92.63** | 81.34 |

16 项细分指标中 **9 项排名第一** **[S1]**：

| 指标 | 得分 | 说明 |
|------|------|------|
| 物体分类 | **100.00** | 满分 |
| 人物动作 | **100.00** | 满分 |
| 时间闪烁 | **99.87** | — |
| 背景一致性 | **98.57** | — |
| 运动平滑度 | **97.99** | — |
| 主体一致性 | **97.68** | — |

### 3.3 评测局限性说明

VBench 为**自动化客观评测**，侧重一致性、平滑度、语义对齐等技术维度。它不等同于人类主观的"视觉惊艳度"或"艺术表现力"。VBench 提供了人类偏好注释数据集来验证与人类感知的对齐程度，但最终排名仍由自动指标决定 **[S4]**。

## 四、TeleTron 开源训练框架

以下信息来自 GitHub 仓库 README **[S2]**：

| 维度 | 详情 |
|------|------|
| 定位 | 长上下文多模态 Transformer 训练框架 |
| 许可证 | Apache 2.0 |
| 版本 | 0.1.0（26 次提交） |
| 语言 | Python 94.7%，CUDA 4.4% |

### 4.1 已实现特性 [S2]

- Ulysses 上下文并行
- 张量并行
- AdaLayerNorm / RmsNorm 融合内核
- 分布式多模态编码器

### 4.2 支持模型 [S2]

- HunyuanVideo（含 I2V 微调）
- Wan-2.2
- Wan-2.1（图像转视频）

### 4.3 性能数据 [S2]

- HunyuanVideo 训练加速 **30%+**（通过分布式多模态编码器）
- 支持超过 30 秒 720P 视频片段训练
- 混合并行 + 激活检查点 + 融合 CUDA 内核优化 GPU 显存

### 4.4 环境要求 [S2]

- NVIDIA PyTorch 容器（24.10-py3）
- 示例需 5 GPU + 512GB 共享内存
- 基于 Megatron-LM、Diffusers、HunyuanVideo 构建

## 五、应用场景

以下声明来自中国电信官方发布 **[S5]**：

- **角色一致性**：保证单个或多个主体人物在各视频片段中的外观一致性
- **动作控制**：实现对复杂动作和交互式动作的精确控制
- **物理合规**：让角色和目标物体的运动符合物理规律
- **影视全流程**：整合语义、语音、文生图、文生视频能力，覆盖文字脚本撰写、分镜脚本绘制、视频拍摄及剪辑、配音及音效合成

## 六、已知局限

### 6.1 论文自述局限 [S1]

| 局限 | 说明 |
|------|------|
| 风格多样性不足 | 训练数据偏真实世界场景，艺术化风格支持有限 |
| 动物/复杂物体生成薄弱 | 姿态编码器主要针对人体设计 |
| 复杂场景细节有限 | 多主体密集场景下细节刻画待提升 |

### 6.2 基于公开信息的客观观察

## 八、综合评价

**架构层面**："故事板"两阶段解耦是 VAST 最核心的创新。通过结构化中间表示将语义理解与视觉合成分离，在可控性上建立了相对于端到端方案的结构性优势 **[S1]**。

**评测层面**：VBench 总分 89.71，领先第二名近 6 分，9/16 项第一，数据来自论文公开报告 **[S1]**。但需注意 VBench 为自动化评测，与人类主观感受可能存在偏差 **[S4]**。

**工程层面**：TeleTron 框架已开源（Apache 2.0），支持主流视频生成模型训练，有实际工程价值 **[S2]**。但 VAST 模型本身未开源、无 demo，外部无法独立验证其生成质量。

**商业层面**：央企背景决定了其优先服务政企市场的定位，与 C 端产品（可灵、即梦等）不在同一竞争维度 **[S5]**。
